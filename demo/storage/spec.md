# Storage Demo Spec

## Goal

Demonstrate the LastStack storage path as LLVM IR with deterministic evidence:
- checksum-validated durable header format,
- two-phase commit marker,
- recovery checks that reject invalid or uncommitted state.

## Source of Truth

- Storage runtime IR: `demo/storage/ips.ll`
- Evidence gate: `demo/storage/ips-evidence.sh`
- Build pipeline: `demo/storage/build.sh`

## IPS Header Model

Stored record (`ips_header_t`):
- `magic`
- `version`
- `epoch`
- `value`
- `committed`
- `checksum`

Validation rules on read:
- `magic` and `version` must match expected constants.
- `committed` must be `1`.
- `checksum` must match recomputed checksum.

## Commands

`laststack-ips <file> <command>`:
- `init`: create/reset store to epoch `0`, value `0`.
- `show` / `recover`: read and validate committed state.
- `add <delta>`: perform staged write (`committed=0`), then committed write (`committed=1`).
- `corrupt`: write intentionally uncommitted state for negative-path testing.

## Evidence Requirements

`ips-evidence.sh` must pass all checks:
1. init succeeds
2. add +5 succeeds
3. recover returns epoch 1, value 5
4. add -2 succeeds
5. recover returns epoch 2, value 3
6. corrupt succeeds
7. recover after corrupt fails

The script emits `demo/storage/ips-report.json` and fails closed on any mismatch.

## Build and Run

```bash
cd demo/storage
./build.sh
./run.sh
```

## Outputs

Generated by build:
- `demo/storage/laststack-ips`
- `demo/storage/ips-report.json`
